## 🆕 新功能詳解

### 1. **斜杠命令系統**
所有命令現在都使用現代的 Discord 斜杠命令格式：
```
/ban user:@用戶 reason:理由
/mute user:@用戶 minutes:30 reason:洗屏
```

**優點：**
- 自動完成和選項提示
- 更好的用戶體驗
- 強制性參數檢查
- 原生 Discord 集成

### 2. **事件系統**
自動化的事件處理器：
```javascript
// guildMemberAdd - 新成員加入
// - 自動發送歡迎消息
// - 檢查黑名單並自動封鎖
// - 通知管理員

// guildBanAdd - 成員被封鎖
// - 自動記錄到管理日誌

// guildMemberRemove - 成員離開
// - 自動記錄離開事件
```

### 3. **完整的舉報系統**
```
/report user:@用戶 reason:騷擾 description:詳細描述
```
- 6 種舉報類型
- 自動發送給 mod-logs 頻道
- 為每個舉報生成唯一 ID
- 類型分類和篩選

### 4. **幫助系統**
```
/help                    # 顯示所有命令
/help command:ban        # 顯示特定命令幫助
```
包含：
- 命令用法
- 參數說明
- 權限要求
- 使用例子

### 5. **黑名單子命令**
```
/blacklist view page:1         # 查看黑名單
/blacklist check user:@用戶    # 檢查用戶
/blacklist remove userid:123   # 移除用戶
```

### 6. **日誌過濾**
```
/modlog type:WARN count:20     # 查看最新 20 條警告
/modlog type:BAN count:10      # 查看最新 10 條封鎖
```

## 📝 部署指南

### 本地開發
```bash
# 1. 安裝依賴
npm install

# 2. 設置 .env
cp .env.example .env
# 編輯 .env 並填寫你的 token

# 3. 運行
npm start
```

### Docker 部署
```bash
docker-compose up -d
```

### PM2 生產部署
```bash
pm2 start index.js --name "discord-bot-template"
pm2 save && pm2 startup
```

## 📞 技術支持

**常見問題預案**

Q: 機器人不回復命令？
A: 
1. 檢查機器人是否在線（/help）
2. 檢查 token 是否有效
3. 檢查機器人是否有斜杠命令權限

Q: 日誌沒有記錄？
A:
1. 檢查 `data/` 目錄是否存在
2. 確保機器人有寫入權限
3. 查看控制台錯誤消息

Q: 自動功能不工作？
A:
1. 檢查 Intents 是否啟用
2. 確保 events/ 文件已加載
3. 查看控制台日誌
